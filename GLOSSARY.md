## AI & Machine Learning

**AI (Artificial Intelligence)**
- Computer programs that can learn and make decisions
- Example: A chatbot that answers your questions

**Machine Learning**
- Teaching computers to learn from examples instead of programming every rule
- Example: Showing a computer 1000 cat pictures so it learns to recognize cats

**TensorFlow.js**
- A tool that lets websites use AI directly in your browser
- No server needed - everything runs on your computer

**COCO-SSD**
- An AI system that can identify objects in pictures (cars, people, dogs, etc.)
- Works in real-time through your webcam

**Merlin AI**
- The helpful AI assistant in this project that answers questions and teaches
- Named after the wizard from King Arthur legends

---

## Blockchain & Crypto

**Blockchain**
- A digital record book that many people share and can't be changed
- Like a shared Google Doc that no one can delete or fake

**Wallet (Crypto Wallet)**
- A digital place to store cryptocurrency (digital money)
- Like a bank account, but you control it completely

**Solana / Ethereum / Tron**
- Different types of blockchain systems (like different banks)
- Each has its own cryptocurrency and rules

**SPL Token**
- A type of digital item on the Solana blockchain
- Like a digital trading card or game token

**Cold Storage**
- Keeping your cryptocurrency offline for maximum security
- Like keeping cash in a safe instead of a bank account

**Private Key**
- A secret password that controls your crypto wallet
- NEVER share this - it's like your ATM PIN

**NFT (Non-Fungible Token)**
- A unique digital item that proves you own something digital
- Like a certificate of authenticity for digital art

## Web Development

**HTML**
- The language that creates the structure of web pages
- Like the blueprint of a building

**CSS**
- The language that makes websites look pretty (colors, fonts, layout)
- Like the paint and decoration on a building

**JavaScript**
- The language that makes websites interactive and dynamic
- Like the electricity that makes things work in a building

**API (Application Programming Interface)**
- A way for different programs to talk to each other
- Like a waiter taking your order to the kitchen

**WebSocket**
- A connection that lets websites update instantly without refreshing
- Like a phone call vs. sending letters

**QR Code**
- A square barcode you can scan with your phone camera
- Used for quick links and connections

---

## 3D Graphics

**3D Visualization**
- Showing objects in 3D on your screen (you can rotate and explore them)
- Like a video game environment

**Babylon.js / Three.js**
- Tools for creating 3D graphics in web browsers
- Like Unity or Unreal Engine, but for websites

**Virtual Machine (in 3D context)**
- A 3D representation of a computer or system
- Not an actual computer, just a 3D model

---

## Authentication & Security

**Authentication**
- Proving who you are (like showing ID)
- Usually done with username and password

**OAuth**
- Logging in using another account (like "Sign in with Google")
- You don't need to create a new password

**Firebase**
- A service by Google for user accounts and data storage
- Handles login, passwords, and database for you

**Local Storage**
- Space in your browser where websites can save information
- Data stays on your computer

**Encryption**
- Scrambling data so only authorized people can read it
- Like writing in secret code

**AES-256**
- A very strong encryption method used by governments and banks
- Practically impossible to crack

---

## System Architecture

**Server**
- A computer that serves (sends) web pages and data to users
- Like a library that gives out books

**Client**
- Your computer or phone that requests and displays web pages
- Like you visiting the library

**Static Hosting**
- Websites that don't need a server to run (just files)
- Like PDF documents you can open anytime

**GitHub Pages**
- Free website hosting from GitHub
- Perfect for static websites and projects

**Node.js**
- A way to run JavaScript on servers (not just browsers)
- Lets you build websites with JavaScript

---

## Development Terms

**Repository (Repo)**
- A folder containing all the code for a project
- Like a filing cabinet for code

**Git**
- A system for tracking changes to code
- Like "Track Changes" in Microsoft Word

**Commit**
- Saving a snapshot of your code changes
- Like saving a version of a document

**Pull Request (PR)**
- Asking to add your changes to the main code
- Like submitting homework for review

**Open Source**
- Code that anyone can see, use, and modify
- Free and community-driven

---

## Mobile & Devices

**Responsive Design**
- Websites that work well on phones, tablets, and computers
- Automatically adjusts to screen size

**QR Code Scanning**
- Using your phone camera to read barcodes for quick actions
- Open camera, point at QR code, tap the link

**Mobile-First**
- Designing for phones first, then adapting for computers
- Most people use phones, so start there

**Cross-Device Sync**
- Using multiple devices that stay in sync (like using desktop and phone together)
- Changes on one device appear on all devices

---

## Gaming & Rewards

**Play-to-Earn (P2E)**
- Games where you can earn real money or cryptocurrency
- Like getting paid to play

**Tokens / Gems**
- Digital points or currency in games or apps
- Can sometimes be traded for real money

**Leaderboard**
- A list showing top players or performers
- Like a high score list in arcade games

**Gamification**
- Adding game elements (points, levels, rewards) to non-game activities
- Makes learning and work more fun

---

## Data & Analytics

**Database**
- An organized collection of information
- Like a digital filing system

**SQL**
- A language for asking questions about databases
- Like searching a library catalog

**JSON**
- A simple way to organize and share data
- Looks like: `{"name": "John", "age": 30}`

**CSV**
- A simple spreadsheet format (comma-separated values)
- Can open in Excel

---

## Network & Communication

**WiFi**
- Wireless internet connection
- How your phone and computer connect to the internet without cables

**IP Address**
- A unique number that identifies your computer on the internet
- Like a street address for your device

**Localhost**
- Your own computer (when acting as a server)
- Address: `http://localhost:8000`

**Port**
- A numbered channel for network communication
- Like different phone extensions at a company

---

## Development Tools

**Console / Terminal**
- A text interface for controlling your computer
- Type commands instead of clicking

**Browser Console (F12)**
- A developer tool built into web browsers
- Shows errors, logs, and lets you test code

**IDE (Integrated Development Environment)**
- A fancy text editor for writing code
- Examples: VS Code, Sublime Text

---

## Testing & Debugging

**Bug**
- An error or problem in code
- Named after an actual moth found in an old computer

**Debug**
- Finding and fixing bugs
- Like being a detective for code problems

**Test**
- Code that checks if other code works correctly
- Like checking your homework before turning it in

**Console Log**
- Messages that developers print to see what's happening
- Like sticky notes for debugging

---

## Performance Terms

**Load Time**
- How long it takes for a website to fully display
- Should be under 3 seconds for good experience

**FPS (Frames Per Second)**
- How smooth animation or video is
- 60 FPS is very smooth, 30 FPS is acceptable

**Optimization**
- Making code faster and more efficient
- Like finding shortcuts to get work done quicker

**Caching**
- Storing data temporarily for faster access later
- Like keeping frequently used items on your desk

---

## Miscellaneous

**Legacy Code**
- Old code that still works but might be outdated
- Like an old but reliable car

**Deprecated**
- Features that are old and will be removed eventually
- Still works but shouldn't be used in new projects

**MVP (Minimum Viable Product)**
- The simplest version that works
- Get it working first, make it fancy later

**Sandbox**
- A safe testing environment where mistakes don't matter
- Like a practice area before going live
